# MIMO Work UI Plugins

MIMO Work supports safe, declarative UI plugin packages for alternate visual assets. A plugin is a directory containing `manifest.json` plus referenced image files. JavaScript, HTML, executable files, and arbitrary CSS are not copied into the MIMO Work data directory.

## Manifest

```json
{
  "id": "starlight",
  "name": "星夜 MIMO Work",
  "version": "1.0.0",
  "author": "MIMO Work Team",
  "description": "深紫星夜配色的 MIMO Work 形象,含主题色与进行中文案。",
  "figures": {
    "swim": "img/bird.png",
    "surf": "img/surf.png",
    "greet": "img/greet.png",
    "sleep": "img/sleep.png",
    "sit": "img/sit.png",
    "toggleIcon": "img/greet.png"
  },
  "labels": {
    "zh": {
      "working": "巡航中…",
      "workingSprint": "流星冲刺中…",
      "workingDive": "潜入星海中…",
      "workingSurf": "星浪滑行中…"
    },
    "en": {
      "working": "Stargazing…",
      "workingSprint": "Meteor dash…",
      "workingDive": "Deep diving…",
      "workingSurf": "Riding stardust…"
    }
  },
  "tokens": {
    "light": {
      "--ds-accent": "#7a5fd0"
    }
  },
  "features": {
    "cameos": true
  }
}
```

## Rules

- `id` must be 2-40 lowercase letters, numbers, or hyphens. Reserved ids: `default`, `kun`, `ikun`, `on`, `off`, `none`.
- Image paths must stay inside the plugin directory and use `png`, `webp`, `jpg`, `jpeg`, or `gif`.
- Each image is limited to 2 MB; all plugin images together are limited to 24 MB.
- Theme tokens are limited to the `--ds-*` allowlist generated by the app.
- Missing figure slots fall back to the default MIMO Work visuals.

## Example

See `examples/ui-plugins/starlight/manifest.json` for a complete package.
