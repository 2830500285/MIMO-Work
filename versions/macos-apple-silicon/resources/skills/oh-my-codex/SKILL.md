# Oh My Codex Workflow

Use this skill when the user asks for a Codex-like coding experience or when work benefits from tight agent execution controls.

## Interaction Pattern

- Show progress while working: reading files, editing files, running commands, waiting for model/tool output, and verifying.
- Keep permission mode visible and understandable: ask for approval, approve for me, full access, or custom.
- Keep reasoning effort explicit: auto, low, medium, high, extra high.
- Support review, terminal, browser, files, and side chat as first-class workspace actions.
- Prefer local, scoped environment variables over global proxy or credential changes.

## UI Behavior

- Primary action stays in the composer.
- Utility actions live near the composer or top-right environment menu.
- Menus should use icons, clear labels, and short descriptions.
- Color theme is product-owned; copy Codex interaction structure without copying its visual palette.
